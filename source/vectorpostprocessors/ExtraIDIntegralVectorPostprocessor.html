ExtraIDIntegralVectorPostprocessor

Integrates or averages variables based on extra element IDs

Overview

The ExtraIDIntegralVectorPostprocessor object is a vector postprocessor to integrate or average input variables or material properties based on multiple extra element integer IDs. First, it finds unique combinations of extra IDs, and then it computes separate integral values over input variables and material properties for elements with these unique combinations. For reactor applications, component-wise values such as pin-by-pin power distribution can be easily tallied using this object when the mesh contains the appropriate IDs.

The ExtraIDIntegralVectorPostprocessor object needs the following parameter:

  • "id_name": list of extra IDs by which to separate integrals

The following parameters are optional, but at least one of them should be specified:

  • "variable": variables that this VectorPostprocessor operates on.

  • "mat_prop": material properties that this VectorPostprocessor operates on.

Additionally, a user can control whether volume-integrated values or volume-averaged values are computed by setting the following parameter:

  • "average": whether or not to compute volume average.

commentnote:Vector names / CSV output column names

ExtraIDIntegralVectorPostprocessor declares a vector for each variable or material property, named after the variable or material property, holding the values of the integrals computed.

[VectorPostprocessors<<<{"href": "../../syntax/VectorPostprocessors/index.html"}>>>]
  [integral]
    type = ExtraIDIntegralVectorPostprocessor<<<{"description": "Integrates or averages variables based on extra element IDs", "href": "ExtraIDIntegralVectorPostprocessor.html"}>>>
    variable<<<{"description": "The names of the variables that this VectorPostprocessor operates on"}>>> = 'value1'
    id_name<<<{"description": "List of extra element ID names by which to separate integral(s)."}>>> = 'assembly_id'
  []
[]
(moose/test/tests/vectorpostprocessors/extra_id_integral/extra_id_vpp.i)

Example use to output a field of the integral values

The ExtraIDIntegralVectorPostprocessor object can also be used as a functor for providing spatial values of the integrated or averaged values. A FunctorAux could then be used to output the local values of this functor (the integrals) as an auxiliary elemental variable. The variable has element-wise constant values. The value of the integral with each unique combination of extra ids is output to all the elements sharing that same combination. When the ExtraIDIntegralVectorPostprocessor object is used as a functor, "spatial_value_name" must be set to indicate which variable or material property is to be used.

commentnote

When the ExtraIDIntegralVectorPostprocessor object is used as a functor, for example in FunctorAux, "force_preaux" should set to true, otherwise the value of this object would likely be lagged in the functor evaluation because user objects defining functors are not put into the pre_aux group automatically by MOOSE.

[VectorPostprocessors<<<{"href": "../../syntax/VectorPostprocessors/index.html"}>>>]
  [eeid]
    type = ExtraIDIntegralVectorPostprocessor<<<{"description": "Integrates or averages variables based on extra element IDs", "href": "ExtraIDIntegralVectorPostprocessor.html"}>>>
    id_name<<<{"description": "List of extra element ID names by which to separate integral(s)."}>>> = id
    variable<<<{"description": "The names of the variables that this VectorPostprocessor operates on"}>>> = u
    force_preaux<<<{"description": "Forces the UserObject to be executed in PREAUX"}>>> = true
    #spatial_value_name = u
  []
[]

[AuxVariables<<<{"href": "../../syntax/AuxVariables/index.html"}>>>]
  [v]
    family<<<{"description": "Specifies the family of FE shape functions to use for this variable"}>>> = MONOMIAL
    order<<<{"description": "Specifies the order of the FE shape function to use for this variable (additional orders not listed are allowed)"}>>> = CONSTANT
    [AuxKernel]
      type = FunctorAux
      functor = eeid
      execute_on = timestep_end
    []
  []
[]
(moose/test/tests/vectorpostprocessors/extra_id_integral/functor_test.i)
