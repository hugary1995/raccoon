SpatialUserObjectVectorPostprocessor

Outputs the values of a spatial user object in the order of the specified spatial points

Overview

This VectorPostprocessor evaluates a spatial user object at a set of points and sets each postprocessor to the spatial value of the user object at each point. This postprocessor may be used to convert spatial user objects such as NearestPointLayeredAverage to a postprocessor that can be transferred to another application with a MultiAppVectorPostprocessorTransfer.

If neither "points" or "points_file" are provided, this object will attempt to get the spatial points directly from the user object. These points will represent locations in space where the user object obtains a unique value. To use this feature, the user object must define the spatialPoints() interface.

commentnote:Vector names / CSV output column names

SpatialUserObjectVectorPostprocessor declares a vector with its own name. The full reporter name ends up being <vpp_object_name>/<vpp_object_name>.

Example Input File Syntax

[UserObjects<<<{"href": "../../syntax/UserObjects/index.html"}>>>]
  [npla]
    type = NearestPointLayeredAverage<<<{"description": "Computes averages of a variable storing partial sums for the specified number of intervals in a direction (x,y,z). Given a list of points this object computes the layered average closest to each one of those points.", "href": "../userobjects/NearestPointLayeredAverage.html"}>>>
    direction<<<{"description": "The direction of the layers."}>>> = x
    points<<<{"description": "Computations will be lumped into values at these points."}>>> = '0.5 0.25 0.25
              0.5 0.75 0.25
              0.5 0.25 0.75
              0.5 0.75 0.75'
    num_layers<<<{"description": "The number of layers."}>>> = 3
    variable<<<{"description": "The name of the variable that this object operates on"}>>> = u
  []
[]

[VectorPostprocessors<<<{"href": "../../syntax/VectorPostprocessors/index.html"}>>>]
  [vpp]
    type = SpatialUserObjectVectorPostprocessor<<<{"description": "Outputs the values of a spatial user object in the order of the specified spatial points", "href": "SpatialUserObjectVectorPostprocessor.html"}>>>
    userobject<<<{"description": "The userobject whose values are to be reported"}>>> = npla
    points_file<<<{"description": "A filename that should be looked in for points. Each set of 3 values in that file will represent a Point.  This and 'points' cannot be both supplied."}>>> = points.txt
  []
[]
(moose/test/tests/vectorpostprocessors/spatial_userobject_vector_postprocessor/spatial_userobject.i)
