ADMatchedValueBC

Implements a NodalBC which equates two different Variables' values on a specified boundary.

Description

ADMatchedValueBC is a ADNodalBC which applies to systems of two or more variables, and can be used to impose equality of two solutions along a given boundary. This class is appropriate for systems of partial differential equations (PDEs) of the form where is the domain, and is its boundary, , are the unknowns, , are forcing functions (which may depend on both and ), and and are given fluxes. The v parameter is used to specify the variable whose value is tied to . In the example below, the other variable's name happens to be v as well.

Example Input Syntax

  [left_u]
    type = ADMatchedValueBC
    variable = u
    boundary = 3
    v = v
  []
(moose/test/tests/bcs/ad_matched_value_bc/test.i)

# Solves a pair of coupled diffusion equations where u=v on the boundary

[Variables]
  [u]
    order = FIRST
    family = LAGRANGE
    initial_condition = 3
  []

  [v]
    order = FIRST
    family = LAGRANGE
    initial_condition = 2
  []
[]

[Kernels]
  [diff_u]
    type = ADDiffusion
    variable = u
  []

  [diff_v]
    type = ADDiffusion
    variable = v
  []
[]

[BCs]
  [right_v]
    type = ADDirichletBC
    variable = v
    boundary = 1
    value = 3
  []

  [left_u]
    type = ADMatchedValueBC
    variable = u
    boundary = 3
    v = v
  []
[]

[Executioner]
  type = Steady
  solve_type = 'NEWTON'

  nl_rel_tol = 1e-10
  l_tol = 1e-12
[]

[Outputs]
  file_base = out
  exodus = true
[]
