LowerBoundNodalKernel

Used to prevent a coupled variable from going below a lower bound

Description

LowerBoundNodalKernel is meant to be used to impose a lower bound on a coupled variable v. The variable specified by the variable parameter is a Lagrange multiplier. It should have an order equivalent to that of the coupled variable v. The lower bound is specified using the lower_bound parameter. By default it is 0. An optional parameter is exclude_boundaries which can be used to specify boundary nodes where the LowerBoundNodalKernel should not be applied. This can be useful for avoiding singularities in the preconditioning matrix that can arise when a constraint is active in the same place that another constraint (like a DirichletBC is also active).

Example Syntax

[NodalKernels<<<{"href": "../../syntax/NodalKernels/index.html"}>>>]
  [positive_constraint]
    type = LowerBoundNodalKernel<<<{"description": "Used to prevent a coupled variable from going below a lower bound", "href": "LowerBoundNodalKernel.html"}>>>
    variable<<<{"description": "The name of the variable that this residual object operates on"}>>> = lm
    v<<<{"description": "The coupled variable we require to be greater than the lower bound"}>>> = u
    exclude_boundaries<<<{"description": "Boundaries on which not to execute the NodalKernel. This can be useful for avoiding singuarility in the matrix in case a constraint is active in the same place that a DirichletBC is set"}>>> = 'left right'
  []
  [forces]
    type = CoupledForceNodalKernel<<<{"description": "Adds a force proportional to the value of the coupled variable", "href": "CoupledForceNodalKernel.html"}>>>
    variable<<<{"description": "The name of the variable that this residual object operates on"}>>> = u
    v<<<{"description": "The coupled variable which provides the force"}>>> = lm
  []
[]

  • search_methodnearest_node_connected_sidesChoice of search algorithm. All options begin by finding the nearest node in the primary boundary to a query point in the secondary boundary. In the default nearest_node_connected_sides algorithm, primary boundary elements are searched iff that nearest node is one of their nodes. This is fast to determine via a pregenerated node-to-elem map and is robust on conforming meshes. In the optional all_proximate_sides algorithm, primary boundary elements are searched iff they touch that nearest node, even if they are not topologically connected to it.
