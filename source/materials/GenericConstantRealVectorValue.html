GenericConstantRealVectorValue

Object for declaring a constant 3-vector as a material property.

warningwarning

This object is deprecated in factor of the GenericConstantVectorMaterial

commentnote

If you are working with a std::vector<double>-valued material property, e.g. a variable-size vector, you have to use the GenericConstantStdVectorMaterial.

Overview

GenericConstantRealVectorValue creates a RealVectorValue material property that use constant values to fill the value.

This can be used to quickly create simple constant 3-vector properties, for testing, for initial survey of a problem or simply because the material properties do not vary much over the domain explored by the simulation.

The AD counterpart ADGenericConstantRealVectorValue creates a ADRealVectorValue material property.

Example Input File Syntax

[Materials<<<{"href": "../../syntax/Materials/index.html"}>>>]
  [vector]
    type = GenericConstantRealVectorValue<<<{"description": "Object for declaring a constant 3-vector as a material property.", "href": "GenericConstantRealVectorValue.html"}>>>
    vector_name<<<{"description": "Name of the vector material property to be created"}>>> = constant
    vector_values<<<{"description": "Values defining the constant vector"}>>> = '1 2 3'
    outputs<<<{"description": "Vector of output names where you would like to restrict the output of variables(s) associated with this object"}>>> = all
  []
[]

In the default nearest_node_connected_sides algorithm, primary boundary elements are searched iff that nearest node is one of their nodes. This is fast to determine via a pregenerated node-to-elem map and is robust on conforming meshes. In the optional all_proximate_sides algorithm, primary boundary elements are searched iff they touch that nearest node, even if they are not topologically connected to it. This is more CPU-intensive but is necessary for robustness on any boundary surfaces which has disconnections (such as Flex IGA meshes) or non-conformity (such as hanging nodes in adaptively h-refined meshes).
