QuadraturePointsPositions

Positions of element quadrature points.

Overview

The QuadraturePointsPositions may be block-restricted to limit the gathering of quadrature points to elements from certain subdomains.

This object is currently set to:

  • using the default element order for the quadrature

  • using the Gauss quadrature rule

  • using the element quadrature

commentnote

The QuadraturePointsPositions, when used in conjunction with a TransientMultiApp enables creating the same subapps as the QuadraturePointMultiApp, except that it is not limited to transients.

Input Parameters

  • blockThe list of blocks (ids or names) that this object will be applied

    C++ Type:std::vector<SubdomainName>

    Controllable:No

    Description:The list of blocks (ids or names) that this object will be applied

  • initial_positionsPositions at the beginning of the simulation

    C++ Type:PositionsName

    Controllable:No

    Description:Positions at the beginning of the simulation

  • quadrature_orderAUTOOrder of the volumetric quadrature. If unspecified, defaults to the local element default order. This is not the problem default, which is based on the order of the variables in the system.

    Default:AUTO

    C++ Type:MooseEnum

    Options:AUTO, CONSTANT, FIRST, SECOND, THIRD, FOURTH, FIFTH, SIXTH, SEVENTH, EIGHTH, NINTH, TENTH, ELEVENTH, TWELFTH, THIRTEENTH, FOURTEENTH, FIFTEENTH, SIXTEENTH, SEVENTEENTH, EIGHTTEENTH, NINTEENTH, TWENTIETH

    Controllable:No

    Description:Order of the volumetric quadrature. If unspecified, defaults to the local element default order. This is not the problem default, which is based on the order of the variables in the system.

  • quadrature_typeGAUSSType of the quadrature rule

    Default:GAUSS

    C++ Type:MooseEnum

    Options:CLOUGH, CONICAL, GAUSS, GRID, MONOMIAL, SIMPSON, TRAP, GAUSS_LOBATTO

    Controllable:No

    Description:Type of the quadrature rule
