MultiAppPositions

Obtain positions from MultiApps. This may only be used to set the positions of those same multiapps if an 'initial_positions' parameter is used.

Overview

MultiApps have a position that is used to translate their frame of reference compared to the main app frame of reference. This position can be used to align the meshes for the transfers or for outputting the MultiApps Exodus results at the translated positions.

warningwarning

This should not be used to generate new MultiApps. The MultiApps have already been created when the positions are gathered.

Example MultiApp File Syntax

In this example, the MultiAppPositions is obtaining the positions of three MultiApps.

[Positions<<<{"href": "../../syntax/Positions/index.html"}>>>]
  [multiapp]
    type = MultiAppPositions<<<{"description": "Obtain positions from MultiApps. This may only be used to set the positions of those same multiapps if an 'initial_positions' parameter is used.", "href": "MultiAppPositions.html"}>>>
    multiapps<<<{"description": "Name(s) of the multiapps providing the positions"}>>> = 'm1 m2 m3'
  []
[]

[MultiApps<<<{"href": "../../syntax/MultiApps/index.html"}>>>]
  [m1]
    type = CentroidMultiApp<<<{"description": "Automatically generates Sub-App positions from centroids of elements in the parent app  mesh.", "href": "../multiapps/CentroidMultiApp.html"}>>>
    input_files<<<{"description": "The input file for each App.  If this parameter only contains one input file it will be used for all of the Apps.  When using 'positions_from_file' it is also admissable to provide one input_file per file."}>>> = 'multiapp_positions.i'
    cli_args<<<{"description": "Additional command line arguments to pass to the sub apps. If one set is provided the arguments are applied to all, otherwise there must be a set for each sub app."}>>> = "MultiApps/active='';Positions/active=''"
  []
  [m2]
    type = FullSolveMultiApp<<<{"description": "Performs a complete simulation during each execution.", "href": "../multiapps/FullSolveMultiApp.html"}>>>
    input_files<<<{"description": "The input file for each App.  If this parameter only contains one input file it will be used for all of the Apps.  When using 'positions_from_file' it is also admissable to provide one input_file per file."}>>> = 'multiapp_positions.i'
    cli_args<<<{"description": "Additional command line arguments to pass to the sub apps. If one set is provided the arguments are applied to all, otherwise there must be a set for each sub app."}>>> = "MultiApps/active='';Positions/active=''"
  []
  [m3]
    type = FullSolveMultiApp<<<{"description": "Performs a complete simulation during each execution.", "href": "../multiapps/FullSolveMultiApp.html"}>>>
    input_files<<<{"description": "The input file for each App.  If this parameter only contains one input file it will be used for all of the Apps.  When using 'positions_from_file' it is also admissable to provide one input_file per file."}>>> = 'multiapp_positions.i'
    positions<<<{"description": "The positions of the App locations.  Each set of 3 values will represent a Point.  This and 'positions_file' cannot be both supplied. If this and 'positions_file'/'_objects' are not supplied, a single position (0,0,0) will be used"}>>> = '0.2 0.4 1
                 0 0 6'
    cli_args<<<{"description": "Additional command line arguments to pass to the sub apps. If one set is provided the arguments are applied to all, otherwise there must be a set for each sub app."}>>> = "MultiApps/active='';Positions/active=''"
  []
[]
(moose/test/tests/positions/multiapp_positions.i)

Input Parameters

  • multiappsName(s) of the multiapps providing the positions

    C++ Type:std::vector<MultiAppName>

    Controllable:No

    Description:Name(s) of the multiapps providing the positions

Required Parameters

  • initial_positionsPositions at the beginning of the simulation

    C++ Type:PositionsName

    Controllable:No

    Description:Positions at the beginning of the simulation

  • use_apps_centroidFalseWhether to use the mesh centroid offset by the app position rather than just the position of each child app

    Default:False

    C++ Type:bool

    Controllable:No

    Description:Whether to use the mesh centroid offset by the app position rather than just the position of each child app

    Controllable:No

    Description:The list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.
