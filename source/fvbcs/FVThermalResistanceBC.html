warningwarning

The thermal resistance concept does not apply to conducting slabs with heat sources or systems undergoing transients. This boundary condition should only be applied to steady-state or pseudo-steady transients.

commentnote

The emissivity parameter represents the emissivity of the _surface_, or last layer, of the conducting slabs.

Example input syntax

In this example, we set thermal resistance boundary conditions on the top and left boundaries. We specify emissivity, thermal conductivity and convective heat transfer coefficients to model those three heat transfer mechanisms where present (they are set to 0 when absent).

The inner iteration parameters are exposed for the left boundary. This is only meant to be used to fine-tune a simulation for performance, or if convergence difficulties are encountered for that boundary.

[FVBCs<<<{"href": "../../syntax/FVBCs/index.html"}>>>]
  [left]
    type = FVThermalResistanceBC<<<{"description": "Thermal resistance Heat flux boundary condition for the fluid and solid energy equations", "href": "FVThermalResistanceBC.html"}>>>
    geometry<<<{"description": "type of geometry"}>>> = 'cartesian'
    variable<<<{"description": "The name of the variable that this boundary condition applies to"}>>> = u
    T_ambient<<<{"description": "constant ambient temperature"}>>> = 10
    htc<<<{"description": "heat transfer coefficient"}>>> = 'htc'
    emissivity<<<{"description": "emissivity of the surface"}>>> = 0.2
    thermal_conductivities<<<{"description": "vector of thermal conductivity values used for the conduction layers"}>>> = '0.1 0.2 0.3'
    conduction_thicknesses<<<{"description": "vector of conduction layer thicknesses"}>>> = '1 0.7 0.2'
    boundary<<<{"description": "The list of boundary IDs from the mesh where this object applies"}>>> = 'left'

    # Test setting iteration parameters
    step_size<<<{"description": "underrelaxation step size"}>>> = 0.02
    max_iterations<<<{"description": "maximum iterations"}>>> = 120
    tolerance<<<{"description": "tolerance to converge iterations"}>>> = 1e-4
  []
  [top]
    type = FVThermalResistanceBC<<<{"description": "Thermal resistance Heat flux boundary condition for the fluid and solid energy equations", "href": "FVThermalResistanceBC.html"}>>>
    geometry<<<{"description": "type of geometry"}>>> = 'cartesian'
    variable<<<{"description": "The name of the variable that this boundary condition applies to"}>>> = u
    # Test setting the temperature separately from the variable
    temperature<<<{"description": "temperature variable"}>>> = 'u'
    T_ambient<<<{"description": "constant ambient temperature"}>>> = 14
    htc<<<{"description": "heat transfer coefficient"}>>> = 'htc'
    emissivity<<<{"description": "emissivity of the surface"}>>> = 0
    thermal_conductivities<<<{"description": "vector of thermal conductivity values used for the conduction layers"}>>> = '0.1 0.2 0.3'
    conduction_thicknesses<<<{"description": "vector of conduction layer thicknesses"}>>> = '1 0.7 0.4'
    boundary<<<{"description": "The list of boundary IDs from the mesh where this object applies"}>>> = 'top'
  []
  [other]
    type = FVDirichletBC<<<{"description": "Defines a Dirichlet boundary condition for finite volume method.", "href": "FVDirichletBC.html"}>>>
    variable<<<{"description": "The name of the variable that this boundary condition applies to"}>>> = u
    boundary<<<{"description": "The list of boundary IDs from the mesh where this object applies"}>>> = 'right bottom'
    value<<<{"description": "value to enforce at the boundary face"}>>> = 0
  []
[]

  • search_methodnearest_node_connected_sidesChoice of search algorithm. All options begin by finding the nearest node in the primary boundary to a query point in the secondary boundary. In the default nearest_node_connected_sides algorithm, primary boundary elements are searched iff that nearest node is one of their nodes. This is fast to determine via a pregenerated node-to-elem map and is robust on conforming meshes. In the optional all_proximate_sides algorithm, primary boundary elements are searched iff they touch that nearest node, even if they are not topologically connected to it. This is more CPU-intensive but is necessary for robustness on any boundary surfaces which has disconnections (such as Flex IGA meshes) or non-conformity (such as hanging nodes in adaptively h-refined meshes).
