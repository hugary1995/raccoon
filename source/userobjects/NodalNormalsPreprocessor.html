NodalNormalsPreprocessor

An object that prepares MOOSE for computing nodal normal vectors. This object is automatically created via the [NodalNormals] input block.

The NodalNormalsPreprocessor populates the nodal_normal_x, nodal_normal_y and nodal_normal_z variables with the local quadrature weight times the gradient of the shape function. This object is created by the by AddNodalNormalsAction for the boundaries specified in the "boundary" parameter. See NodalNormals System for more information.

Input Parameters

  • surface_boundaryThe list of boundary IDs where nodal normals are computed

    C++ Type:std::vector<BoundaryName>

    Unit:(no unit assumed)

    Controllable:No

    Description:The list of boundary IDs where nodal normals are computed

Required Parameters

  • blockThe list of blocks (ids or names) that this object will be applied

    C++ Type:std::vector<SubdomainName>

    Unit:(no unit assumed)

    Controllable:No

    Description:The list of blocks (ids or names) that this object will be applied

  • corner_boundaryNode set ID which contains the nodes that are in 'corners'.

    C++ Type:BoundaryName

    Unit:(no unit assumed)

    Controllable:No

    Description:Node set ID which contains the nodes that are in 'corners'.

  • execute_onTIMESTEP_ENDThe list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.

    Default:TIMESTEP_END

    C++ Type:ExecFlagEnum

    Unit:(no unit assumed)

    Options:NONE, INITIAL, LINEAR, NONLINEAR_CONVERGENCE, NONLINEAR, POSTCHECK, TIMESTEP_END, TIMESTEP_BEGIN, MULTIAPP_FIXED_POINT_END, MULTIAPP_FIXED_POINT_BEGIN, FINAL, CUSTOM

    Controllable:No

    Description:The list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.

    Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    C++ Type:bool

    Unit:(no unit assumed)

    Controllable:No

    Description:For the old and older state use projected material properties interpolated at the quadrature points. To set up projection use the ProjectedStatefulMaterialStorageAction.
